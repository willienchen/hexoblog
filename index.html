手動將 Unity 輸出為 XCode Framework

發表於 2019-06-19 更新於 2023-11-16 分類於 Unity

這個課題困擾了我很久，終於成功後
馬上做個紀錄！

首先 將 Unity 輸出 iOS Project
建立 Cocoa Touch Framework Project
建立 Unity Group
將 Untiy Export Project 內的 Library , Classes 複製到 剛剛建立的 Unity Group 底下
以 Create group 方式 加入專案
將 Untiy Export Project 內的 Data 複製到 剛剛建立的 Unity Group 底下
以 Create reference 方式 加入專案
刪除 reference : Unity/Libraries/libil2cpp

將不用編譯之文件從 target 移除

main.mm
DynamicLibEngineAPI-functions.h
DynamicLibEngineAPI.mm

調整 Build Setting

Build Setting Value
Header Search Paths $(PROJECT)/Unity/Classes
$(PROJECT)/Unity/Classes/Native
$(PROJECT)/Unity/Libraries/bdwgc/include
$(PROJECT)/Unity/Libraries/libil2cpp/include

-fno-strict-overflow
-DRUNTIME_IL2CPP=1
-DNET_4_0
Other Linker Flags -weak_framework
CoreMotion
-weak-lSystem
Prefix Header $(PROJECT)/Unity/Classes/Prefix.pch

解決使用 Unity 2019 有太多 Warring

“No previous prototype for function” ,
“This function declaration is not a prototype” ,
“Code will never be executed”

調整以下參數忽略掉

Build Setting Value
Missing Function Prototypes No
Strict Prototypes No
Unreachable Code No

使用 Lunchctl 啟動 unity cache server

發表於 2019-05-22 更新於 2023-11-16

今天在安裝 unity cache server

目標能夠在 mac os 的 server 上自動執行
所以研究了一下 launchctl

unity cache server 是使用 node.js 運行的

在寫 plist 之前 ，必須先找到 node.js 和 unity cache server 的路徑

之後建立一個 plist 檔案

Label：顯示的名稱
ProgramArguments：運行的命令,全部組合起來就是

StandardErrorPath : Error Log 的位置
StandardOutPath：Log 位置

之後將 plist 放入 /Library/LaunchDaemond 底下運行就可以了
